# 支持关系构建

确保权利要求得到说明书的充分支持。

## 支持性要求

1. 权利要求中的每个技术特征都要在说明书中有对应描述
2. 说明书的实施例要能够支撑权利要求的保护范围
3. 上位概念要有足够的下位实施例支撑

## 检查要点

- 权利要求的技术特征 → 说明书有详细描述
- 独立权利要求的范围 → 实施例覆盖边界情况
- 从属权利要求的特征 → 说明书有对应实施例

## 工具使用

- `support_checker` - 检查权利要求与说明书的支持关系
