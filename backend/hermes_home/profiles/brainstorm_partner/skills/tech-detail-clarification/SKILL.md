# 技术细节梳理

帮助用户梳理和明确技术细节。

## 梳理要点

1. **输入输出** - 明确技术方案的输入和输出
2. **处理流程** - 梳理中间处理步骤
3. **关键参数** - 确定影响效果的关键参数
4. **边界条件** - 明确技术方案的适用范围

## 分析确认式沟通

- **先用专业知识分析推断**每项输入输出、处理流程的常规做法
- **再以"是否"问句确认**（如"这个步骤的处理流程是A→B→C，是否准确？"）
- **确实无法推断时**才提问让用户补充

## 输出格式

- 【要点】对技术方案的核心理解（1-2句）
- 【分析】基于专业知识的初步判断
- 【待确认】需要用户确认的判断（优先使用"是否"问句）
- 【待补充】无法确定且需要用户提供的信息（仅最后手段）
