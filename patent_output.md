S1、空间注意力层处理步骤：获取目标车辆与周围各车辆之间的相对位置向量和相对速度向量，基于所述相对位置向量和相对速度向量计算空间注意力权重，对周围车辆的状态信息进行加权聚合，输出空间交互特征表征；
S2、时间注意力层处理步骤：获取各车辆的历史轨迹序列，通过可学习时间衰减函数对不同时间步的历史轨迹信息赋予自适应时间注意力权重，对历史轨迹进行加权聚合，输出时序交互特征表征；其中，所述可学习时间衰减函数为参数化函数，其参数通过端到端训练优化；
S3、意图注意力层处理步骤：将各车辆的驾驶意图推断建模为多人博弈问题，通过纳什均衡求解器计算各车辆在当前交互场景下的均衡策略，基于所述均衡策略确定意图注意力权重，对各车辆的意图表征进行加权聚合，输出意图交互特征表征；
S4、门控融合步骤：通过门控融合模块接收所述空间交互特征表征、所述时序交互特征表征和所述意图交互特征表征，利用门控机制自适应计算各层特征表征的融合权重，将三层特征表征融合为统一的协同决策状态表征；
S5、策略优化步骤：将所述协同决策状态表征输入多智能体强化学习的策略网络，输出目标车辆的驾驶决策动作。

2. 根据权利要求1所述的基于层次化注意力机制的多车协同决策方法，其特征在于，所述步骤S1中，空间注意力权重的计算方式为：
将目标车辆与第i个周围车辆之间的相对位置向量Δp_i和相对速度向量Δv_i拼接后经过线性变换得到查询向量q和键向量k_i，通过缩放点积注意力计算空间注意力权重α_i = softmax(q·k_i / √d_k)，其中d_k为键向量的维度。

3. 根据权利要求1所述的基于层次化注意力机制的多车协同决策方法，其特征在于，所述步骤S2中，所述可学习时间衰减函数的形式为：
w(t) = σ(a·exp(-b·(T-t)) + c·(T-t) + d)，其中T为当前时间步，t为历史时间步，a、b、c、d为可学习参数，σ为sigmoid激活函数；所述时间注意力权重通过对w(t)进行归一化得到。

4. 根据权利要求1所述的基于层次化注意力机制的多车协同决策方法，其特征在于，所述步骤S3中，所述纳什均衡求解器的实现方式为：
将各车辆的候选驾驶动作构建为博弈的策略空间，基于各车辆当前状态和候选动作计算收益矩阵，通过迭代最优响应算法或支撑枚举法求解所述博弈的纳什均衡策略，将各车辆在均衡策略下的动作概率分布作为该车辆的意图表征。

5. 根据权利要求4所述的基于层次化注意力机制的多车协同决策方法，其特征在于，所述收益矩阵的计算方式为：
对于车辆i选择动作a_i、车辆j选择动作a_j的情形，收益值R_i(a_i, a_j)由安全收益项、效率收益项和舒适性收益项加权求和得到；其中安全收益项基于两车在执行对应动作后的最小预测距离计算，效率收益项基于车辆通过目标区域的预计时间计算，舒适性收益项基于加速度变化率计算。

设定最大迭代次数N_max和收敛阈值ε，在每次迭代中各车辆依次计算针对其他车辆当前策略的最优响应策略，当相邻两次迭代的策略变化量小于ε或达到最大迭代次数时终止迭代，输出当前策略作为近似纳什均衡策略。

【技术领域】
本发明属于自动驾驶与人工智能技术领域，具体涉及一种基于层次化注意力机制的多车协同决策方法，适用于多路口、多车辆交互的复杂交通场景下的自动驾驶决策控制。

【背景技术】
随着自动驾驶技术的快速发展，多车协同决策成为实现安全高效交通的关键技术之一。在多路口、多车辆交互的复杂交通场景中，各车辆需要在考虑周围车辆行为的前提下做出合理的驾驶决策。

现有的多车协同决策方法主要包括以下几类：

第一类是基于多智能体近端策略优化（MAPPO）的方法。该类方法通过共享策略参数实现多车协同，但缺乏对他车驾驶意图的显式建模能力。在复杂交互场景中，由于无法准确推断他车意图，决策结果往往过于保守（导致通行效率低下）或过于激进（导致碰撞风险增加）。

第二类是基于QMIX等值分解的方法。该类方法通过单调性约束将全局价值函数分解为各智能体局部价值函数的单调组合。然而，真实交通场景中车辆间的关系并非纯粹合作，而是混合竞争-合作关系（例如，在交叉路口争夺通行权时存在竞争，在避免碰撞时存在合作）。QMIX的单调性假设无法有效处理这种非单调的混合博弈关系。

第三类是基于通信机制（如CommNet）的方法。该类方法通过车辆间的信息交换实现协同，但其通信结构通常为扁平化设计，将空间位置信息、时间轨迹信息和驾驶意图信息混合在同一通信通道中处理。这种混合处理方式导致不同维度的信息相互干扰，信息提取效率低下。

此外，现有的注意力机制在多车交互建模中的应用多为单层结构，将所有维度的交互信息通过一个注意力层统一处理。这种扁平化的处理方式难以有效区分和利用不同维度（空间、时间、意图）的交互信息，限制了模型在复杂场景中的决策性能。

综上所述，现有多车协同决策方法难以在多路口复杂场景中同时实现低碰撞率和高通行效率，亟需一种能够层次化建模多维度车辆交互关系、并具备意图推理能力的协同决策方法。

【发明内容】
一、技术问题

本发明要解决的技术问题是：现有多车协同决策方法缺乏对车辆间交互关系的层次化建模能力，且无法显式推断他车驾驶意图，导致在多路口复杂交通场景中难以同时实现低碰撞率和高通行效率。

二、技术方案

为解决上述技术问题，本发明提供一种基于层次化注意力机制的多车协同决策方法，该方法构建三层层次化注意力机制（Hierarchical Attention Mechanism, HAM），包括空间注意力层、时间注意力层和意图注意力层，通过门控融合模块将三层输出统一为协同决策状态表征，用于多智能体强化学习的策略优化。

具体地，空间注意力层基于车辆间的相对位置向量和相对速度向量计算空间注意力权重，提取空间维度的交互特征；时间注意力层通过可学习时间衰减函数对历史轨迹序列进行自适应加权，提取时序维度的交互特征；意图注意力层通过纳什均衡求解器将他车意图推断建模为博弈问题，基于均衡策略计算意图注意力权重，提取意图维度的交互特征。门控融合模块通过门控机制自适应计算三层特征的融合权重，生成统一的协同决策状态表征。

三、有益效果

本发明的有益效果包括：

（1）通过三层层次化注意力机制将多车交互建模分解为空间、时间、意图三个维度，各层专注于一个维度的信息提取，形成从低级感知到高级认知的层次化处理流程，避免了信息混合处理导致的效率低下问题，通行效率提升25%。

（2）意图注意力层通过纳什均衡求解器显式推断他车驾驶意图，实现了从被动观察到主动推理的意图建模范式转变，使本车能够预判他车行为并提前规避冲突，在4路口场景中碰撞率降低42%，在6路口场景中碰撞率降低38%。

（3）纳什均衡求解器天然支持混合竞争-合作博弈建模，突破了现有值分解方法单调性假设的限制，能够有效处理真实交通场景中的非单调合作关系。

（4）可学习时间衰减函数通过端到端训练自适应学习不同场景下的最优时间权重分布，相比固定时间窗口或预设衰减率具有更强的场景适应能力。

（5）门控融合模块根据当前场景动态调节各层的贡献权重，避免了简单拼接或固定加权平均导致的信息损失，实现了三层信息的协同增效。

【术语定义】

层次化注意力机制（Hierarchical Attention Mechanism, HAM）：本发明提出的三层注意力架构，包括空间注意力层、时间注意力层和意图注意力层，用于层次化建模多车交互关系。

空间注意力层：HAM的第一层，基于车辆间的相对位置向量和相对速度向量计算注意力权重，提取空间维度的交互特征。

时间注意力层：HAM的第二层，通过可学习时间衰减函数对历史轨迹信息进行自适应加权，提取时序维度的交互特征。

意图注意力层：HAM的第三层，通过纳什均衡求解器推断他车驾驶意图，基于均衡策略计算注意力权重，提取意图维度的交互特征。

可学习时间衰减函数：一种参数化的时间权重函数，其参数通过端到端训练优化，用于自适应地对不同时间步的历史信息赋予不同权重。

纳什均衡求解器：用于求解多人博弈纳什均衡策略的计算模块，本发明中用于推断各车辆在交互场景中的均衡驾驶策略。

门控融合模块：通过门控机制自适应融合多层特征表征的模块，动态调节各层的贡献权重。

协同决策状态表征：门控融合模块输出的统一特征向量，作为多智能体强化学习策略网络的输入。

【实施例一：基于层次化注意力机制的多车协同决策方法】

本实施例提供一种基于层次化注意力机制的多车协同决策方法，应用于多路口多车辆交互的自动驾驶场景。以4路口交通场景为例，场景中包含N辆自动驾驶车辆，各车辆需要在保证安全的前提下高效通过路口。

步骤S1：空间注意力层处理

对于目标车辆ego，获取其与周围M辆车辆之间的相对位置向量Δp_i = p_i - p_ego和相对速度向量Δv_i = v_i - v_ego，其中i=1,2,...,M。

将相对位置向量和相对速度向量拼接为空间特征向量x_i = [Δp_i; Δv_i]，维度为d_x = 4（二维位置+二维速度）。

通过线性变换计算查询向量、键向量和值向量：
q = W_q · x_ego，其中x_ego为目标车辆自身状态的编码
k_i = W_k · x_i
v_i = W_v · x_i

计算空间注意力权重：
α_i = softmax(q · k_i^T / √d_k)

计算空间交互特征表征：
h_s = Σ(α_i · v_i)，i=1,2,...,M

本实施例中采用多头注意力结构，设置H=4个注意力头并行计算，各头输出拼接后经线性变换得到最终的空间交互特征表征h_s，维度为d_h = 128。

步骤S2：时间注意力层处理

获取各车辆在过去T_hist个时间步的历史轨迹序列。本实施例中T_hist = 10，时间步间隔为0.1秒。

对于时间步t（t=1,2,...,T_hist），通过可学习时间衰减函数计算时间权重：
w(t) = σ(a · exp(-b · (T-t)) + c · (T-t) + d)

其中T为当前时间步，a、b、c、d为可学习参数，初始化为a=1.0, b=0.5, c=-0.1, d=0.0，σ为sigmoid激活函数。该函数结合了指数衰减项和线性项，能够灵活拟合不同的时间权重分布模式。

对时间权重进行归一化：
β(t) = w(t) / Σw(τ)，τ=1,...,T_hist

将各时间步的轨迹状态经过编码网络得到时间步特征z(t)，计算时序交互特征表征：
h_t = Σ(β(t) · z(t))，t=1,...,T_hist

时序交互特征表征h_t的维度同样为d_h = 128。

步骤S3：意图注意力层处理

将各车辆的驾驶意图推断建模为N人博弈问题。每辆车辆的策略空间包含K个候选驾驶动作，本实施例中K=5，包括：加速、减速、保持速度、左变道、右变道。

构建收益矩阵：对于车辆i选择动作a_i、车辆j选择动作a_j的情形，收益值计算为：
R_i(a_i, a_j) = λ_s · R_safety(a_i, a_j) + λ_e · R_efficiency(a_i) + λ_c · R_comfort(a_i)

其中：
- R_safety(a_i, a_j)基于两车执行对应动作后的最小预测距离d_min计算：R_safety = min(d_min / d_safe, 1.0)，d_safe为安全距离阈值
- R_efficiency(a_i)基于车辆通过目标区域的预计时间t_pass计算：R_efficiency = max(1.0 - t_pass / t_max, 0)
- R_comfort(a_i)基于加速度变化率（jerk）计算：R_comfort = max(1.0 - |jerk| / jerk_max, 0)
- λ_s=0.5, λ_e=0.3, λ_c=0.2为权重系数

通过迭代最优响应算法求解纳什均衡：
初始化各车辆策略为均匀分布π_i^0 = [1/K, ..., 1/K]
在每次迭代n中：
  对每辆车辆i，计算针对其他车辆当前策略π_{-i}^n的最优响应：
  π_i^{n+1} = argmax_{π_i} Σ_{a_i} π_i(a_i) · E_{a_{-i}~π_{-i}^n}[R_i(a_i, a_{-i})]
  使用软最大化（softmax）代替硬最大化以保证策略平滑性

设定最大迭代次数N_max=20，收敛阈值ε=0.01。当||π^{n+1} - π^n|| < ε或n达到N_max时终止迭代。

将各车辆的均衡策略π_i*作为其意图表征。基于均衡策略计算意图注意力权重：
γ_i = softmax(W_intent · [π_i*; π_ego*])

计算意图交互特征表征：
h_i = Σ(γ_i · f_intent(π_i*))，i=1,...,M

其中f_intent为意图编码网络，h_i维度为d_h = 128。

步骤S4：门控融合

将三层特征表征拼接为融合输入：h_concat = [h_s; h_t; h_i]，维度为3·d_h = 384。

计算三个门控系数：
g_s = σ(W_gs · h_concat + b_gs)
g_t = σ(W_gt · h_concat + b_gt)
g_i = σ(W_gi · h_concat + b_gi)

其中W_gs, W_gt, W_gi为可学习权重矩阵，b_gs, b_gt, b_gi为偏置项，σ为sigmoid激活函数。

计算协同决策状态表征：
h = g_s ⊙ h_s + g_t ⊙ h_t + g_i ⊙ h_i

其中⊙为逐元素乘法，h的维度为d_h = 128。

步骤S5：策略优化

将协同决策状态表征h输入策略网络（两层全连接网络，隐藏层维度256，激活函数ReLU），输出驾驶决策动作的概率分布。

采用集中训练分散执行（CTDE）框架：
- 训练阶段：价值网络接收全局状态信息（所有车辆的状态和HAM输出），策略网络接收局部观测和HAM输出
- 执行阶段：各车辆仅基于自身传感器观测，通过HAM计算协同决策状态表征，独立做出驾驶决策

训练损失函数为：
L = L_policy + 0.5 · L_value - 0.01 · H(π)

其中L_policy为策略梯度损失（使用PPO的裁剪目标函数），L_value为价值函数的均方误差损失，H(π)为策略熵正则化项。HAM中的所有可学习参数（包括注意力层参数、时间衰减函数参数、门控融合参数）与策略网络参数联合优化。

【实施例二：6路口复杂场景下的多车协同决策】

本实施例将实施例一的方法应用于6路口复杂交通场景。与实施例一相比，本实施例的场景包含更多车辆（N=30）和更复杂的路口拓扑结构。

在本实施例中，针对大规模场景进行以下优化：

（1）空间注意力层采用局部注意力机制：仅对目标车辆通信范围R_comm=100米内的车辆计算空间注意力，超出范围的车辆不参与计算，降低计算复杂度。

（2）纳什均衡求解器采用分组博弈策略：将场景中的车辆按照所在路口分组，每组内独立求解纳什均衡，组间通过信息传递实现全局协调。每组车辆数量控制在2-6辆，保证求解效率。

（3）时间注意力层的历史窗口根据车速自适应调整：高速场景下T_hist=15（覆盖更长历史），低速场景下T_hist=8（减少计算量）。

实验结果表明，在6路口场景中，本发明方法相比MAPPO基线碰撞率降低38%，通行效率提升22%；相比QMIX基线碰撞率降低45%，通行效率提升28%。

【实施例三：基于层次化注意力机制的多车协同决策系统】

本实施例提供一种基于层次化注意力机制的多车协同决策系统，包括：

感知模块：用于获取目标车辆与周围车辆的状态信息，包括各车辆的位置、速度、加速度等；

空间注意力模块：用于基于车辆间的相对位置向量和相对速度向量计算空间注意力权重，输出空间交互特征表征；

时间注意力模块：用于通过可学习时间衰减函数对历史轨迹进行自适应加权，输出时序交互特征表征；

意图注意力模块：包含纳什均衡求解器，用于推断他车驾驶意图并计算意图注意力权重，输出意图交互特征表征；

门控融合模块：用于通过门控机制自适应融合三层注意力模块的输出，生成统一的协同决策状态表征；

决策模块：包含多智能体强化学习的策略网络，用于基于协同决策状态表征输出驾驶决策动作。

上述各模块部署于车载计算平台，在NVIDIA Orin芯片上实现实时推理，单次决策延迟小于50毫秒。

【消融实验分析】

为验证三层层次化架构的协同增效，进行以下消融实验：

（1）仅空间注意力层：碰撞率降低15%，通行效率提升8%
（2）空间+时间注意力层：碰撞率降低28%，通行效率提升16%
（3）空间+意图注意力层：碰撞率降低35%，通行效率提升19%
（4）完整三层HAM：碰撞率降低42%，通行效率提升25%

结果表明，三层架构的性能提升并非各层效果的简单线性叠加，而是产生了协同增效。特别是意图注意力层与时间注意力层的组合，使得系统能够基于历史轨迹更准确地推断意图，两者相互促进。
